**Access Levels** control who can view, modify, or interact with threads and their data. Threadify uses a role-based permission system that provides fine-grained control over thread access.

### Example Usage

<Tabs>
<Tab title="Node.js">
```javascript
// Invite external party
const invitation = await thread.inviteParty({
  role: 'logistics',
  expiresIn: '48h'
});

// External party joins with token
const logisticsThread = await connection.join(invitation.token);
await logisticsThread.step('shipment_created')
  .addContext({ trackingNumber: '1Z999AA10123456784' })
  .success();

// Internal service joins directly
const warehouseThread = await connection.join('thread-uuid', 'warehouse_manager');
```
</Tab>
</Tabs>

### Best Practices

- **Principle of least privilege** - Grant minimum necessary access
- **Role-based access** - Use roles instead of individual permissions
- **Regular audits** - Review and remove unnecessary access
- **Temporary access** - Grant time-limited access when needed
